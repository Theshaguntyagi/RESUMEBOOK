Project 01 — Production AI System
Envigo Auto-Proposal Generator
🚀 Live at Envigo · Fully Automated · Client-Facing
An end-to-end Python pipeline that reads client intake data from Google Forms/Sheets, enriches it with OpenAI GPT, and generates branded Word document proposals — zero human writing required.
The Problem: Envigo's consultants were spending 2–3 hours manually writing each client proposal — personalising the copy, formatting the document, embedding branding assets. This was a direct bottleneck on revenue capacity.

The Solution: A fully automated pipeline where a client fills a Google Form → the data lands in Google Sheets → a background Python process detects new rows, calls OpenAI to rewrite and enhance each answer into professional proposal language, then populates a reusable .docx template and saves the branded proposal to Google Drive — ready to send within minutes.
  • Google Sheets ingestion via gspread + service account (credentials.json). Reads form responses row by row with incremental processing tracked in processed_rows.json to avoid re-running.
  • OpenAI content enhancement using openai.ChatCompletion.create (v0.28 syntax). Each field gets a targeted prompt that transforms raw client answers into polished consultancy prose.
  • python-docx template engine: placeholder replacement across paragraphs, tables, and headers. Special XML run-merging logic handles split runs that broke naive find-replace.
  • Google Drive API logo embedding: downloads the client's logo via Drive API (not raw HTTP requests, which failed on auth-gated files) and inserts it as an image at the correct template position.
  • Error handling & recovery: graceful JSON parse failures, OpenAI retry logic, and Drive API fallback keep the pipeline running unattended.

Project 05 — Game Development
CatchMaster — HTML5 Arcade Game
🎮 Zero Dependencies · AI-Adaptive · Pure JavaScript
A zero-dependency HTML5 Canvas arcade game with an AI-adaptive difficulty engine and a miss-streak mechanic — all in vanilla JavaScript, running directly in any browser without a build step.
The Design Philosophy: Most browser games feel static — the difficulty never adapts to you. CatchMaster uses a real-time adaptive AI system that continuously observes player performance and tunes the game's parameters to keep the player in a "flow state" — challenging but not frustrating.

Core Systems:
  • Adaptive Difficulty Engine: tracks a rolling hit/miss ratio over the last N frames. If the player is catching everything easily → spawns more objects, increases fall speed, shrinks catch radius. If miss rate spikes → eases up. Parameters update smoothly without jarring jumps.
  • Miss-Streak Mechanic: consecutive misses trigger visual and audio cues (screen flash, shake effect) and accelerate game-over conditions — adds tension and stakes beyond a simple lives counter.
  • Canvas 2D Rendering: all game objects (falling items, catcher, particles, score UI) drawn via Canvas 2D API with requestAnimationFrame for a locked 60fps loop. No WebGL, no libraries.
  • Particle System: catch events spawn a burst of colour-coded particles that decay over time — coded as a lightweight object pool to avoid garbage collection pauses mid-game.
  • High Score Persistence: scores persisted to localStorage with a leaderboard view showing top 5 sessions, date-stamped.

Project 06 — Medical AI / Computer Vision
Diabetic Retinopathy Detector
👁️ 92% Accuracy · CNN · Clinical Decision Support
A deep learning model that classifies the severity of diabetic retinopathy from retinal fundus photographs, achieving 92% accuracy — built as a B.Tech final year project with real clinical dataset training.
Clinical Context: Diabetic Retinopathy (DR) is the leading cause of blindness in working-age adults worldwide. Early detection via retinal imaging is critical but requires specialist ophthalmologists — a bottleneck in under-resourced healthcare settings. This project demonstrates that a CNN can automate the screening step to support clinical workflows.

Model Architecture & Training:
  • Dataset: trained on retinal fundus images with 5-class DR severity labels (No DR, Mild, Moderate, Severe, Proliferative DR). Applied class-weighting to handle the severe class imbalance in real clinical datasets.
  • CNN Architecture: custom convolutional architecture in PyTorch with 6 conv blocks, batch normalisation, dropout regularisation, and global average pooling before the classifier head — designed to balance depth with the limited dataset size.
  • Preprocessing Pipeline: fundus images resized and normalised, with CLAHE (Contrast Limited Adaptive Histogram Equalisation) applied to enhance the visibility of microaneurysms and haemorrhages before model input.
  • Data Augmentation: random horizontal/vertical flips, rotation, brightness jitter — critical for a medical dataset where genuine variation is limited and overfitting risk is high.
  • Results: 92% accuracy on the held-out test set. Confusion matrix analysis showed strongest performance on No-DR and Proliferative-DR classes; model struggles most with the Mild/Moderate boundary — consistent with published literature.
  • Grad-CAM Visualisation: implemented gradient-weighted class activation maps to visually verify that the model attends to clinically relevant regions (optic disc, blood vessels) rather than image artefacts.
